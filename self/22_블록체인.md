# 블록체인

## BitCoin White Paper

- 백서(白書) : 정치, 외교, 경제 따위의 각 분야에 대한 현상을 분석하고 미래를 전망하여 그 내용을 알리기 위하여 만든 보고서.
- P2P(Peer-to-Peer) : 신용, 인증 기관 없이 인터넷에서 개인과 개인의 직접 연결 된 거래방식
- 노드 : 네트워크를 구성하는 객체. 블록 만들기에 참여하는 컴퓨터, 참여자
- 블록체인
  - 블록을 체인으로 묶어놓은 저장방식
  - 블록(block) : 데이터를 저장하는 단위. 책으로 치면 한 페이지
  - 체인(chain) : 말 그대로 블록들을 계속 엮어놓은 것
- 포크 : 갈림길이 갈라지듯이 블록체인이 둘로 나뉘어지는 현상
  - 한 블록에서 해시함수로 이어진 블록이 두개 이상일 때 발생
  - 포크가 일어나지 않기 위해 필요한 '합의 알고리즘'
- 이중지불(double-spending) : 동일한 자산이 두 명의 수신자(payee)에게 동시에 전송될 수 있는 문제
- 타임스탬프(timestamp) : 특정한 시각을 나타내는 문자열. 
  - 전자문서가 특정한 시점에 존재하고 있었다는 '존재증명'과 그 이후 데이터가 변경되지 않았다는 '내용증명'을 해주는 일종의 전자 도장 역할
- 인센티브
  - 블록을 생성, 투표서비스를 제공하며 대가를 받는 것
  - 작업증명을 하기 위해 작업을 하는 행위(채굴)를 통해 받게되는 보상
- 비트코인 : 분산 공개 원장
- 비잔틴 오류 허용(Byzantine Fault Tolerance. BFT)
  - 정직한 참여자끼리 합의에 도달할 수 있다
  - 바이러스, 네트워크 등 불안정한 노드가 들어와도 전체시스템이 진행된다

## Ethereum White Paper

### 초록

- 블록체인 기술을 이용한 대안적 어플리케이션들
  - 컬러드 코인(colored coins) : 사용자 정의화폐와 금융상품을 블록체인 위에 표현
    - 비트코인 블록체인을 통해 현물자산을 디지털 형태로 표현하는 자산 발행 레이어
    - 오픈 애셋 프로토콜(Open Asset Protocol)
    - 비트코인 블록체인과 연결되어 해당 블록체인 위에서 누구나 발행할 수 있는 암호화폐의 성격
    - 비트코인 블록체인과 현물자산을 연결하는 기술을 가진다면 컬러드 코인이라고 할 수 있다.
    - 사용자는 특정 비트코인 UTXO에 공개적으로 색깔을 부여함으로써 새로운 화폐를 발행 할 수 있다.
  - 스마트 자산(smart property) : 물리적 대상의 소유권을 표현
  - 네임코인(Namecoin) : 도메인 이름과 같은 비동질적(non-fungible) 자산을 기록
  - 스마트 컨트랙트(smart contracts) : 임의적인 계약규칙을 구현한 코드에 의해 디지털 자산을 관리하는 좀 더 복잡한 형태
  - 자율조직(decentralized autonomous organizations. DAOs) : 블록체인을 기반으로 한 탈중앙화된 자율 조직
- 튜링완전 프로그래밍 언어가 심어진 블록체인
- 코딩 규칙에 따라 '어떤 상태'를 다르게 변환시키는 기능(arbitrary state transition functions)이 포함 된 계약(contracts)을 **유저들이 작성할 수 있게 함**

### 역사

- 분산화된 디지털 통화의 개념은 비단 블록체인이 나오기 이전부터 나오던 것이다.
- 80~90년대 익명 e-cash 프로토콜
  - 차우미안 블라인딩(Chaumian blinding)으로 알려진 로우레벨 암호 알고리즘에 기반
  - 개인정보를 강력하게 보호하는 화폐 제공
  - 하지만 중앙집권적인 중개인(3rd trusted party)에 의존하는 한계
- 1998년 'Wei Dai'의 b-money
  - 분산합의와 계산 퍼즐을 풀게한 화폐 발행
  - 분산합의구현에 대한 방법 제시를 하지 못함
- 2005년 'Hall Finney'의 재사용 가능한 작업증명(reusable proofs of work)
  - b-money아이디어에 Adam back의 '계산 난이도 해시캐시 퍼즐'을 조합한 것
  - 외부의 신뢰를 필요로 하는 컴퓨팅(trusted computing)을 기반에 둠으로써 이상 구현에 실패
- 2009년 사토시 나카모토 bitcoin
  - 공개키 암호방식을 통한 소유권 관리를 위해 사용되던 기존의 알고리즘을 '작업 증명(proof of work)'이라고 알려진 합의 알고리즘과 결합
  - 실제적으로 구현된 탈중앙화된 화폐
- 작업증명기반의 작동방식의 효과
  - 간단하면서 효과적인 합의 알고리즘
    - 모든 노드들이 비트코인의 장부상태에 일어난 표준 업데이트의 집합(a set of canonical updates)에 공동으로 동의할 수 있도록 해줌
  - 누구나 합의 프로세스에 참여가 가능
    - 합의결정권에 대한 정치적 문제를 해결
    - 시빌공격(sybil attacks)도 방어해줄 수 있는 매커니즘
    - 합의 프로세스에 대한 참여 조건으로 '특정한 리스트에 등록된 주체이어야만 한다'는 어떤 형식적 장벽대신에 경제적 장벽(각 노드의 결정권의 크기를 그 노드의 계산능력에 직접적으로 비례시킴)으로 대체하는 것
- 지분증명(proof of stake)의 등장
  - 각 노드의 계산능력이 아닌 화폐의 보유량에 따라 각 노드의 결정권 정도를 계산해야 한다는 것

---

- 비트코인 = 암호화 화폐의 장부 = 상태변환시스템(state transition system)
  - 현재 모든 비트코인의 소유권 현황으로 이루어진 하나의 "상태(state)"와 트랜잭션을 받아서 그 결과로써 새로운 상태를 출력해주는 "상태변환함수(state transition function)"로 구성
  - 상태는 모든계좌잔고표(balance sheet)
  - 트랜잭션은 A에서 B로 $X를 송금하라는 요청
  - 상태변환함수에 의해 A의 계좌에서는 $X가 감소, B의 계좌에서는 $X가 증가
- "상태(state)"는 생성되었지만 아직 사용되지 않은 모든 코인들의 집합(소비되지 않은 트랜잭션출력. Unspent Transactions Outputs. UTXO)
  - 각자의 코인금액이 표시
  - UTXO의 소유자(20byte의 주소로 정의되는 암호화된 공개키)정보
- UTXO(Unspent Transactions Outputs)
  - 아직 소비되지 않은 트랜재션 결과물
  - 미지출 거래 출력
  - 비트코인 블록체인에 기록된 소비되지 않은 출력값을 통해 거래의 유효성을 검사하여 코인의 존재 여부를 확인한다.
  - 비교) 이더리움의 '계좌 잔고 모델(Account Balance Model)'
- 트랜잭션은 하나 이상의 입력 및 출력을 포함한다
  - 입력
    - 보내는 쪽(송신자) 지갑주소에서 선택된 기존 UTXO에 대한 참조정보
    - 해당지갑주소에 대응되는 개인키
    - 이 두가지를 합쳐서 생성한 암호화된 서명
  - 출력
    - 상태에 추가될 새로운 UTXO정보

#### 블록체인 기술을 이용한 다른 응용 사례(Alternative Blockchain Applications)

- 05년 'Nick Szabo'의 "소유주 권한을 통한 재산권 보장"
  - 정주(homesteading), 불법점유, 지공주의 개념사용
  - 누가 어떤 땅을 가지고 있느냐에 대한 문제를 블록체인 기반 시스템으로 처리할 수 있다
  - 데이터베이스 복제 기술. 파일 복제 시스템의 부재로 실패
- 09년 분권 합의 시스템 발전
- 네임코인
  - 2010년 만들어진 네임코인
  - 탈중앙화된 명칭 등록 데이터베이스
  - 사용자가 타인과 서로 교류하기 위해서 각자의 계정을 필요로 하는데 이를 의사난수 해쉬가 아닌 "george"와 같은 이름으로 구별하는 것
  - 선출원주의로 먼저 등록한 사람이 성공
- 컬러드코인
  - 누구나 비트코인 블록체인 위에서 **자신만의 고유한 디지털 화폐를 발행**할 수 있는 프로토콜 역할을 하는 것.
  - 자기 자신만의 디지털 토큰을 발행하는 프로토콜
  - 사용자가 특정 UTXO에 공개적으로 색깔을 부여함으로써 새 화폐를 "발행"
  - 다른 UTXO 색깔은 이미 소비된 것으로 간주하는 거래의 입력과 같은 색깔이 되도록
  - 블록체인을 역추적해 그들이 받은 UTXO의 색깔을 정함으로써, 사용자가 특정 색깔을 가진 UTXO만 집갑에 간직하고 그 코인을 보통 비트코인처럼 여기저기 보낼 수 있게 한다.
- 메타코인
  - 비트코인 거래를 메타코인 거래 저장에 이용
  - 상태 이동 함수 APPLY를 다르게 가짐으로써 비트코인 시스템 위에서 운영되는 프로토콜을 갖는 것

#### 스크립팅(Scripting)

- 비트코인에 구현된 스크립트 언어의 몇가지 한계
  - 튜링불완전성
    - 모든 경우의 프로그래밍을 다 지원하지는 않는다.
    - while이나 for와 같은 순환(loop) 명령 카테고리가 빠져있다.
      - 왜? 거래 증명을 할 때 무한 순환에 빠지는 것을 막기 위해서
  - 가치 무지
    - UTXO 스크립트만으로는 인출 액수를 세밀하게 통제할 방법이 없다.
    - UTXO는 인출액 전부가 송금되거나 말거나 밖에 선택할 수가 없다. 즉, 세부 작은 단위로 나눠질 가능성을 포함할 수 없는 것
  - 상태표현제한
    - UTXO가 표현할 수 있는 상태는 사용되었거나 안 되거나 둘 뿐
    - 이 두가지 상태 이외에 다른 어떤 내부적 상태를 가지는 다중 단계 계약이나 스크립트를 만들 수가 없다.
    - 즉 UTXO는 단순하고 1회적인 계약에만 이용될 수 있을 뿐, 분산조직과 같은 더 복잡한 "상태적(stateful)" 계약에는 이용될 수 없고 메타프로토콜을 적용하기 어렵게 만든다
  - 블록체인 무지(Blockchain-)

## Mastering Etereum

### 3. 이더리움 클라이언트

- 이더리움 사양을 구현하고 다른 이더리움 클라이언트와 피어투피어 네트워크를 통해 통신하는 SW 어플리케이션
- 서로 다른 이더리움 클라이언트들끼리 상호운용(interoperate)이 가능

- 동일한 이더리움 네트워크에서 운영 및 상호작용하는 데 사용될 수 있다

- 이더리움 네트워크

- 이더리움 클래식, 엘라, 익스팬스 ...
- 이더리움 프로토콜의 기본 구현
  - 러스트(Rust)로 작성된 패리티(Parity)
  - 고(Go)로 작성된 게스(Geth)
  - C++로 작성된 cpp-ethereum
  - 파이썬(Python)으로 작성된 pyethereum
  - 스칼라(Scala)로 작성된 맨티스(Mantis)
  - 자바(Java)로 작성된 하모니(Harmony)
- 노드설정, 커맨드라인 및 API탐색 방법

- 이더리움 개발에는 라이브 메인넷 네트워크에서 실행 중인 풀노드가 필요하지 않다

- 게스(Go-Ethereum, Geth)
  - 고(Go)언어로 개발 된 이더리움 '공식' 클라이언트
  - 일반적으로 모든 이더리움 기반 블록체인은 자체 게스 구현을 갖고 있기 때문에 게스를 실행하고 있다면, 적합한 버전을 찾아야 한다.

- 이더리움 클라이언트

  - 애플리케이션 인터페이스
  - JSON(JavaScript Object Notation)으로 인코딩된 RPC(Remote Procedure Call)명령을 제공
  - JSON-RPC-API를 참고하여 확인 가능
    - 이더리움 클라이언트를 이더리움 네트워크 및 블록체인의 게이트웨이(gateway)로 사용하는 프로그램을 작성할 수 있게 해주는 인터페이스
    - RPC 인터페이스는 포트 8545에서 HTTP 서비스로 제공

  - 요청에 포함되는 4가지 요소
    - jsonrpc : JSON-RPC 프로토콜의 버전
    - method : 호출할 메서드의 이름
    - params : 메서드를 호출하는 동안 사용할 파라미터값을 보유하는 구조화된 값(생략 가능)
    - id : 클라이언트가 설정한 식별자

- 원격 이더리움 클라이언트

  - 다른 곳에서 실행되고 있는 풀 노드에 연결하여 이더리움 블록체인의 로컬 사본으로 동기화하지 않고 풀 노드 이더리움 클라이언트의 기능 중 일부만 제공

  - 모바일 지갑
    - 잭스, 스태터스, 트러스트월렛, 사이퍼 브라우저

  - 브라우저 지갑
    - 메타마스크, 잭스, 마이이더월렛 ...

## Ethereum강의

- 탈중앙 어플리케이션 제작을 위한 대안 프로토콜 구축이 목적
  - 빠른 개발시간, 보안, 다른 어플리케이션과의 효율적 상호작용 등에 주안
  - **튜링 완전 언어를 내장하고 있는 블록체인**이라는 필수적이고 근본적인 기반 제공
    - 누구나 스마트 계약 및 분산 어플리케이션 작성 지원
    - 소유권에 대한 임의의 규칙, 거래 형식(transaction format), 상태변환 함수(state transition function) 등 생성가능
    - ∴ 매우 간단

## 코인 용어정리

- 손절 : 손해를 보고 코인을 판매하는 행위
- 익절 : 이익을 얻고 코인을 판매하는 행위
- 거래소 : 현금(원화, 달러), 코인(비트코인, 이더리움)으로 코인들을 사고 팔 수 있는 사이트
  - 빗썸, 코인원, 업비트 ...
- 김프(김치 프리미엄) : 해외에 비해 국내외 수요는 특히 많으나 재정거래가 막혀있어서 제한된 코인수로 국내 투자자들이 거래를 하게 되며 발생한 프리미엄
- 재정거래 : 간단하게 외국에서 달러로 저렴하게 코인을 사와서 김프가 많은 국내시장에 판매하는 것

- 평단 : 코인을 구매한 평균금액

- 하락장(떡락) : 코인의 가격이 크게 떨어지는 시장상황
- 상승장(불장, 떡상) : 코인의 가격이 크게 올라가는 시장상황
- 단타 : 투자를 하고 단시간에 코인을 사고팔아 이익을 얻는 행위
- 코인수 늘리기 = 평단 낮추기
  - 가격이 떨어지는 시장상황에서 높은 가격에 팔고, 낮은 가격에 사는 것을 반복함으로써 코인수를 늘리는 행위

- 메인넷 : 이더리움, 네오 같은 **오픈소스 기반의 토큰**이 자체기술력으로 토큰에서 코인으로 나아가는 경우
  - 이더리움 집안에서 태어나서 새집 찾아 떠나는 트론, 이오스

- 상장 : 거래소에서 판매를 시작한다는 의미
  - 트론이 코인베이스에 상장되었다 = 코인베이스에서 판매를 시작했다
- 상폐(상장폐지) : 거래소에서 판매를 중단한다는 의미. 상장폐지가 된다고 코인이 없어지는 것은 아니지만, 시세는 바닥으로 곤두박질 친다.
- 올인(ALL IN) : 자신의 투자금을 다 투자하는 행위
- 사토시 : 비트코인 기술인 블록체인을 처음 논문에 서술한 사토시 교수의 빌려 비트코인 단위량을 말하는 것이다.
  - 0.00000005 비트코인 가격인 토큰이 있다면, 이를 5사토시라고 부른다.
- 비타릭 : 이더리움을 만든 창시자이며, 이더리움 단위량을 말하는 것
  - 0.00000001 = 1비탈릭
- 채굴(마이닝) : 광부들이 광산에서 금을 캐듯, 채굴업자나 개인이 컴퓨터를 이용하여 어려운 산술식을 풀어냄으로서 보상으로 가상화폐를 얻는 행위
- 코인 : 메인넷에 상자된 암호화화폐를 일컫는 말
  - 비트코인, 이더리움, 리플 ...
- 토큰 : 이더리움, 네오 같은 오픈소스를 바탕으로 만들어진 암호화화폐. 알트들이 여기 속한다고 보시면 될 듯
- 스캠 : 사기목적의 코인. 다른 코인의 백서를 복사하거나 기술을 복사하여 이익을 얻고 사기칠려는 스캠
- 백서 : 개발한 코인에 대해 서술한 문서. 개발목적이나 향후계획 등(ICO에서 확인)
- 세력 : 거대자본을 무기로 시세를 조작하는 집단
- 고래 : 거대자본을 가지고 있는 투자자
- 개미 : 일반적인 소액 투자자들
- 개미털기 : 세력을 상승시키고 싶은 코인에서 개미들의 지분을 줄이고자 하는 행위

## 이더리움 용어정리

- 튜링완전(Turing-Complete) : 어떤 프로그래밍 언어나 가상 머신이 튜링머신과 동일한 계산 능력으로 문제를 풀 수 있다는 의미

  - 1936년 "결정 문제에 적용한 연산 가능한 숫자에 관하여"
  - 컴퓨터로 실험자 2명이 서로 대화할 때 누가 사람인지 구분할 수 없다면 컴퓨터는 인공지능을 갖춘 것(튜링 완정성)

  - 튜링 완전 언어 + 무한한 저장공간 = 모든 계산 가능한 문제를 계산해내는 기계 = 튜링 머신(인간의 뇌)
- 튜링완전 프로그래밍 언어
  - 튜링머신과 동일한 계산 능력을 가진다는 의미
  - 모든 계산적인 문제를 프로그래밍 언어로 풀 수 있다는 의미
- 암호화폐 지갑(cryptocurrency wallet)
  - 비트코인, 이더리움 등의 **암호화폐를 보관할 수 있는 계정**
  - 구조
    - 지갑주소(공개키)와 암호로 구성
    - 지갑주소는 다른 사람들이 송금할 수 있도록 공개
    - 개인 암호는 지갑 소유자 본인만 알고있어야 한다
  - 암호화폐 종류별로 따로 지갑을 가지고 있어야 한다
  - 콜드월렛과 핫월렛
    - 인터넷 연결 유무로 나뉘게 된다
    - 온라인 지갑(핫월렛)은 인터넷 주소가 네트워크에 연결되어있어 실시간 거래가 가능하나 보안에 취약
      - 온라인으로 연결되는 모든 지갑
      - 일반적으로 사람들이 사용하는 지갑
      - 실시간으로 편리하게 쓸 수 있지만 해킹, 보안위험에 노출될 가능성
      - 해커는 실시간 거래가 이뤄지는 핫월렛을 공격해 이용자 개인키를 탈취하는 것
    - 오프라인 지갑(콜드월렛, 하드웨어 지갑)은 실시간 거래가 불가능하나 해킹위험에서 벗어나 보안에 강함
      - 거래내역 생성
      - 개인키로 서명
      - 전송
- 스마트 컨트랙트(smart contract)
  - 서면으로 이루어지던 계약을 코드로 구현하고 특정 조건이 충족되었을 때 해당 계약이 이행되게 하는 *script*
  - `94 Nick Szabo에 의해 처음 제안
  - Solidity라는 언어 사용 => 바이트 코드로 변환하여 블록체인에 배포

- 이런 것들의 상위 개념은 디지털 화폐(Digital currency)
  - 디지털 형태로 된 화폐로 동전이나 지폐와 달리 물리적 형태가 없다.
  - 전자 화폐(Electronic currency)라고 불리기도 한다.
- 가상화폐(virtual currency)
  - 실물 없이 네트워크로 연결된 가상 공간에서 전자적 형태로 사용되는 화폐
  - (위키백과) 정부에 의해 통제를 받지 않는 디지털 화폐의 일종으로 개발자가 발행, 관리하며 특정한 가상 커뮤니티에서만 통용되는 결제 수단이라고 정의
- 암호화폐(cryptocurrency)
  - 가상화폐의 일종
  - 암호를 사용해 새로운 코인을 생성하거나 거래를 안전하게 진행할 수 있도록 한 화폐
  - 비트코인을 최초의 암호화폐로 볼 수 있고 이더리움, 라이트코인, 리플 등도 암호화폐다.
  - 블록체인 기술과 맞닿아 있는것이 바로 암호화폐
  - 데이터가 모든 참여자에게 분산 저장돼 누군가 데이터를 위,변조한다고 해도 신뢰성이 깨지지 않는다는 장점
- 가상통화
  - 한국에서 사용하는 가상화폐의 명칭
  - 상품 교환가치의 척도라는 제도적인 의미에서 화폐가 통화보다 강력하다고 보기 때문
  - 화폐가 보다 실제적이고 구체적인 '돈'이라는 개념이 강하다면 통화는 화폐를 포함해 유통이나 지불수단을 전반적으로 지칭하는 추상적인 개념이라는 것
